# 家族旅行スケジュール＆準備物アプリ

家族（父：貴弘／母：友紀／長女：一果／長男：陸）の旅行・出張の
**スケジュール**と**準備物（持ち物チェックリスト）**を管理し、複数人で共有するためのWebアプリ。
PC・iPhoneのブラウザで動作する単一HTMLファイル。

## ファイル構成
- `index.html` … アプリ本体（これ1つで完結）
- `serve.ps1` … ローカル動作確認用の簡易サーバー（公開時は不要）
- `README.md` … このメモ

## 主な機能
- **全体タイムテーブル**：横軸＝日付／縦軸＝時間。種別ごとに色分け。
- **日別ビュー**：1日分を時系列カードで表示。電話番号タップ発信・地図リンク対応。
- **予定の種別**：訪問先・Activity／移動（交通・予約状況）／食事／観光／ホテル（支払有無）／その他。
  - ホテル：ホテル名・住所・電話・チェックイン/アウト・支払・料金
  - 移動：交通手段・便名/列車名・出発/到着（空港・駅・ターミナル）・予約状況・予約番号/座席
  - 訪問先：担当・目的（Activity名/観光など）・住所・電話
  - 食事：店名・特色料理・住所・電話
- **準備物（参加者別）**：参加者ごとに持ち物をチェックボックス管理。進捗バー付き。
  よく使う持ち物（衣服・水着・水遊び道具・サングラス・おむつ等）はワンタップ追加チップで投入可。
  参加者は追加・編集・削除可能。
- **テキスト一括取込**：予約メールや旅程をコピペ→自動で予定化（日付・時刻・種別を推定）。
- **バックアップ**：JSON書き出し／復元、印刷・PDF。

## データの保存と共有
- 既定は**この端末のローカル保存**（localStorage）。設定不要ですぐ使える。
- 「⚙ 共有設定」で **Firebase** を設定すると、家族の全端末で**リアルタイム自動同期**。
  - 同期の単位は「共有ID」（家族で同じ文字列にする）。
  - 設定はアプリ内の手順ガイド参照（Firestoreをテストモードで作成→firebaseConfigを貼付→共有ID入力）。

## ローカルで動作確認する方法
1. `serve.ps1` を右クリック →「PowerShellで実行」
   （または PowerShell で `powershell -ExecutionPolicy Bypass -File serve.ps1`）
2. ブラウザで http://localhost:8731/ を開く
- もしくは `index.html` をダブルクリックでも動くが、Firebase同期はhttp(s)配信が必要。

## 公開（GitHub Pages）手順 ※未実施／確認後に実施
1. GitHubに新規リポジトリを作成（例：`family-trip`）
2. このフォルダの `index.html` をpush（`serve.ps1`は不要）
3. Settings → Pages → Branch: main / (root) で公開
4. 発行されたURLを家族のiPhone/PCで開く。ホーム画面に追加すればアプリ感覚で使える。

## 既知の設計メモ
- file:// 直開きはlocalStorage制約・Firebase同期不可のため、共有運用はPages配信推奨
  （過去アプリの「file://保存/キャッシュ落とし穴」と同じ理由）。
- PowerShell 5.1 は .ps1 の日本語コメントを誤読することがあるため serve.ps1 はASCIIのみ。
